# @webwaka/core — WebWaka OS v4

## Project Overview
This is a TypeScript library package providing shared platform primitives for building SaaS applications on the Cloudflare Workers ecosystem. It is the foundational core for the WebWaka vertical suites (Fintech, Transport, Logistics, etc.).

## Tech Stack
- **Language:** TypeScript 5.x (ESM, ES2022 target)
- **Runtime target:** Cloudflare Workers (`@cloudflare/workers-types`)
- **Framework:** Hono (>=4.0.0, peer dependency)
- **Testing:** Vitest
- **Package Manager:** pnpm
- **Build Tool:** tsc

## Project Structure
```
src/
  core/
    ai/            — AI abstraction engine (vendor-neutral, 3-tier fallback)
    auth/          — JWT auth, CORS, rate-limiting middleware
    billing/       — Platform billing ledger (integer kobo values)
    booking/       — Booking primitives
    chat/          — Chat utilities
    document/      — Document management
    geolocation/   — Geolocation services
    kyc/           — KYC utilities
    logger/        — Centralized platform logger
    notifications/ — Unified SMS/Email (Yournotify + Termii)
    rbac/          — Role-based access control
  index.ts         — Main re-export entry point
dist/              — Compiled output (generated by tsc)
server.mjs         — Minimal status server for Replit preview (port 5000)
```

## Module Exports
- `@webwaka/core` — All primitives
- `@webwaka/core/auth`
- `@webwaka/core/rbac`
- `@webwaka/core/billing`
- `@webwaka/core/logger`

## Scripts
- `pnpm run build` — Compile TypeScript to `dist/`
- `pnpm run test` — Run Vitest test suite
- `pnpm run lint` — Type-check without emitting

## Workflow
The "Start application" workflow runs `pnpm run build && node server.mjs` and serves a status page at port 5000.

## Key Design Principles
- **Nigeria & Africa First:** Financial values stored as integer kobo (not floats)
- **Zero Console Logs:** All logging goes through the platform logger
- **Vendor Neutral AI:** Three-tier fallback (Tenant OpenRouter → Platform OpenRouter → Cloudflare AI)
